简介

Hexo是一个快速、简洁且高效的博客框架。Hexo使用 Markdown（或其他标记语言）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

安装

安装 Hexo 相当简单，只需要先安装下列应用程序即可：

  • Node.js (Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)
  • Git

在命令行中使用npm安装Hexo

安装完成后使用hexo命令测试是否安装成功

网站相关

字段 注解 实例
title 网站标题 title: ‘小李的博客’
subtitle 网站副标题 subtitle: ‘涛之博客’
description 网站描述 description: ‘记录自己的日常’
keywords 关键词 keywords: ‘博客’
author 作者 author: ‘蛇皮小李’
language 网站使用的语言，常见的有 zh-Hans和 zh-CN language: ‘zh-CN’
timezone 网站时区。 默认使用您电脑的时区 timezone: ‘Asia/Shanghai’

Gitee

快速部署

  1. 在gitee新建一个远程仓库用来部署博客项目

  2. 配置Hexo项目根目录下的_config.yml文件中的deploy属性

  3. 配置Hexo项目根目录下的_config.yml文件中的URL属性属性

  4. 在Hexo项目根目录下安装hexo-deployer-git快速部署模块

    检查模块是否安装完成

  5. 使用快速部署模块

  6. 打开码云仓库点击服务，然后选择Gitee Pages，需要实名才能使用

  7. 部署静态页面

    1. 访问生成的网址

    源码部署

    1. 新建gitee仓库

    2. 将本地项目pull到远程仓库中

      • 使用hexo g 将源文件装换成静态资源public

      • 修改.gitignore文件，删除public项

      • 修改配置文件：按照快速部署的方式修改配置文件

    3. 使用Gitee Page部署/public文件夹

    • 选择自己想要部署的分支，默认直接master分支就行

    • /public目录就是静态资源目录，包含所有的前端资源，只需要将这个目录提供给外部访问就行

    • 然后点击生成的域名，就直接映射到静态网页了

    注意

    css没有效果：

    如果采用这种方式部署后，网页没有css，这说明项目路由没对，配置博客配置文件

    以后部署博客

    1. 直接用git bash打开博客目录，
    2. 先hexo clean删除public静态文件以及缓存
    3. 然后使用hexo g生成public
    4. 使用git add .
    5. 使用git commit -m 说明（必须要有说明才能使用git push）
    6. 使用git push
    7. 打开码云 找到博客仓库，找到服务 pages 然后直接点击更新，等待1分钟就部署成功
    8. 如果更换了电脑，直接git clone仓库源码就可以了。也可以直接下载仓库zip
